markdown文件的目录树生成

发表于 2018-06-12 | 分类于 markdown | | 本文阅读量 次
字数统计: 440 | 阅读时长 ≈ 2

最近看 github 上项目，发现很多都有文档文件（README.md），看见里面有目录树，故而整理了一下。至于发文的博客网站是否支持，就得看它们自己的考虑了。简书这里是不支持的，你要想加，可以参考简书自动生成目录小工具。

markdown文件的目录树生成的核心在于锚点。

工具类生成

萝卜青菜，各有所爱。每个人用的 markdown 编辑器不一样，在线，离线，客户端，甚至 txt 。基本每个编辑器都有【目录生成】的功能。我用的 Typora ，我找了一下，段落--内容目录，确实能生成目录树，效果还不错。可是切换成源码一看，只是加了个[TOC]。怀着忐忑的心情，贴到简书里，恭喜你，不支持。当然不是所有能用 markdown 写文章的网站都不支持，我记得有支持的，好像CSDN就支持[TOC]。

当然如果你工具生成目录树的源码是markdown的代码，理论上能支持 markdown 发文章的都可以。这种源码长啥样，看下面的那种方式。
